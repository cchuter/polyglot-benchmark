# Iteration 001: polyglot-go-counter

## Summary

Completed the counter exercise — a test suite design exercise where the solution is `counter_test.go`, not production code.

## What Was Accomplished

1. Analyzed all four implementations to understand their bugs:
   - Impl1: counts lines as newline count only (misses non-newline-terminated lines)
   - Impl2: uses manual ASCII range check instead of `unicode.IsLetter`
   - Impl3: iterates by byte index instead of rune, breaking multi-byte characters
   - Impl4: correct implementation using `range` and `unicode.IsLetter`

2. Verified the existing test suite in `counter_test.go` (written in a previous PR #116):
   - 9 test functions covering edge cases and bug detection
   - All pass on Impl4, all detect bugs in Impl1-3

3. Created .osmi artifacts (GOAL.md, SCOPE.md, plan.md, plan-review.md, changelog)

4. Committed and pushed to `issue-158` branch
